This package only consists of one interface: XUIContainer, which is the top abstract class from which all other containers inherit from. That is to say, it is at the top of the container inheritance hierarchy. Containers in general are components that, well, 'contain' other components. For example a XUIWindow allows for a XUIPanel to be placed inside of it. Without this behavior, GUI programmers would not have many options when creating GUIs.

Based on the XUIContainer there are two other sub types available:

  • Top-level containers
  • Intermediate containers

Top level containers are containers that directly interact with the windowing environment. The types found for top level are:

  • XUIWindow
  • XUICustomDialog
  • XUISaveFileDialog
  • XUIOpenFileDialog
  • XUIBasicDialog

Top level containers can contain intermediate containers and menus. Intermediate containers are containers that are found within top level containers but they still have the ability to act as a container. Intermediate containers like:

  • XUIPanel
  • XUISplitPanel
  • XUITabbedPanel

Each one of these types of components can contain atomic components and composite components. There is a special relationship with XUIPanel and XUISplitPanel and XUITabbedPanel. XUIPanel directly contains atomic components whereas XUISplitPanel and XUITabbedPanel only allow the XUIPanel to be placed inside of it. With that special rule understood you will find a rich support of containment in Purnama Project XUI.

For more information about the XUI API, look at the XUI API guide which is part of this download or if you didn't download it, visit The Purnama Project home page. This guide will give you a much deeper understanding as to what can be done with this API. As well, be sure to look at the examples that accompany this API download as well.
